Key Default Type Description
decode.json-parser.enabled
 true Boolean Optional flag to specify whether to use the Jackson JsonParser to decode json with better performance, true by default.
encode.decimal-as-plain-number
 false Boolean Optional flag to specify whether to encode all decimals as plain numbers instead of possible scientific notations, false by default.
fail-on-missing-field
 false Boolean Optional flag to specify whether to fail if a field is missing or not, false by default.
ignore-parse-errors
 false Boolean Optional flag to skip fields and rows with parse errors instead of failing; fields are set to null in case of errors, false by default.
map-null-key.literal
 "null" String Optional flag to specify string literal for null keys when 'map-null-key.mode' is LITERAL, "null" by default.
map-null-key.mode
 "FAIL" String Optional flag to control the handling mode when serializing null key for map data, FAIL by default. Option DROP will drop null key entries for map data. Option LITERAL will use 'map-null-key.literal' as key literal.
timestamp-format.standard
 "SQL" String Optional flag to specify timestamp format, SQL by default. Option ISO-8601 will parse input timestamp in "yyyy-MM-ddTHH:mm:ss.s{precision}" format and output timestamp in the same format. Option SQL will parse input timestamp in "yyyy-MM-dd HH:mm:ss.s{precision}" format and output timestamp in the same format.
