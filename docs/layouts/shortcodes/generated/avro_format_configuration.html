encoding
 binary

Enum

 The encoding to use for serialization. Binary offers a more compact, space-efficient way to represent objects, while JSON offers a more human-readable option.

Possible values:
  • "binary": Use binary encoding for serialization and deserialization.
  • "json": Use JSON encoding for serialization and deserialization.
timestamp_mapping.legacy
 true Boolean Use the legacy mapping of timestamp in avro. Before 1.19, The default behavior of Flink wrongly mapped both SQL TIMESTAMP and TIMESTAMP_LTZ type to AVRO TIMESTAMP. The correct behavior is Flink SQL TIMESTAMP maps Avro LOCAL TIMESTAMP and Flink SQL TIMESTAMP_LTZ maps Avro TIMESTAMP, you can obtain the correct mapping by disable using this legacy mapping. Use legacy behavior by default for compatibility consideration.
