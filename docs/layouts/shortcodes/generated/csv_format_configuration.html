Key Default Type Description
allow-comments
 false Boolean Optional flag to ignore comment lines that start with "#" (disabled by default); if enabled, make sure to also ignore parse errors to allow empty rows
array-element-delimiter
 ";" String Optional array element delimiter string for separating array and row element values (";" by default)
disable-quote-character
 false Boolean Optional flag to disabled quote character for enclosing field values (false by default) if true, quote-character can not be set
escape-character
 (none) String Optional escape character for escaping values (disabled by default)
field-delimiter
 "," String Optional field delimiter character (',' by default)
ignore-parse-errors
 false Boolean Optional flag to skip fields and rows with parse errors instead of failing; fields are set to null in case of errors
null-literal
 (none) String Optional null literal string that is interpreted as a null value (disabled by default)
quote-character
 """ String Optional quote character for enclosing field values ('"' by default)
write-bigdecimal-in-scientific-notation
 true Boolean Enables representation of BigDecimal data type in scientific notation (default is true). For example, 100000 is encoded as 1E+5 by default, and will be written as 100000 if set this option to false. Note: Only when the value is not 0 and a multiple of 10 is converted to scientific notation.
