Key Default Type Description
auth-token
 (none) String Authentication token for secured Triton servers.
circuit-breaker-enabled
 false Boolean Whether to enable circuit breaker protection. When enabled, the circuit breaker will automatically fail fast when the server is unhealthy, preventing cascading failures and reducing load on the failing server. The circuit breaker implements a three-state model: CLOSED (normal), OPEN (failing fast), and HALF_OPEN (testing recovery). Defaults to false.
circuit-breaker-failure-threshold
 0.5 Double Failure rate threshold (0.0-1.0) that triggers the circuit breaker to open. For example, 0.5 means the circuit will open when 50% of recent requests fail. Requires a minimum of 10 requests before evaluation. Defaults to 0.5 (50%). Only effective when circuit-breaker-enabled is true.
circuit-breaker-half-open-requests
 3 Integer Number of successful test requests required in HALF_OPEN state to close the circuit. If any request fails in HALF_OPEN state, the circuit immediately reopens. Defaults to 3 requests. Only effective when circuit-breaker-enabled is true.
circuit-breaker-timeout
 1 min Duration Duration to keep the circuit breaker in OPEN state before transitioning to HALF_OPEN. In HALF_OPEN state, limited requests are allowed to probe if the server has recovered. Defaults to 60 seconds. Only effective when circuit-breaker-enabled is true.
compression
 (none) String Compression algorithm for request body. Currently only gzip is supported. When enabled, the request body will be compressed to reduce network bandwidth.
connection-pool-keep-alive
 5 min Duration Duration to keep idle connections alive in the pool before closing them. Longer durations reduce connection overhead but may hold resources unnecessarily. Defaults to 5 minutes. Only effective when connection-reuse-enabled is true.
connection-pool-max-idle
 50 Integer Maximum number of idle HTTP connections to keep in the pool. Idle connections exceeding this limit will be closed. Defaults to 50. Only effective when connection-reuse-enabled is true.
connection-pool-max-total
 200 Integer Maximum number of HTTP connections allowed in the connection pool. When the limit is reached, new requests will wait until a connection is released. Defaults to 200. Only effective when connection-reuse-enabled is true.
connection-pool-monitoring-enabled
 false Boolean Whether to enable connection pool metrics monitoring. When enabled, metrics such as active connections, idle connections, and pending requests will be exposed through Flink's metric system. Defaults to false. Only effective when connection-reuse-enabled is true.
connection-reuse-enabled
 true Boolean Whether to enable HTTP connection reuse. When enabled, connections are kept alive and reused for subsequent requests to the same endpoint, reducing connection overhead. Defaults to true.
connection-timeout
 10 s Duration Timeout for establishing a new HTTP connection. This is separate from the request timeout and only applies to the initial TCP connection establishment. Defaults to 10 seconds.
custom-headers
 (none) Map Custom HTTP headers as key-value pairs. Example: 'X-Custom-Header:value,X-Another:value2'
endpoint
 (none) String Full URL of the Triton Inference Server endpoint, e.g., https://triton-server:8000/v2/models. Both HTTP and HTTPS are supported; HTTPS is recommended for production.
flatten-batch-dim
 false Boolean Whether to flatten the batch dimension for array inputs. When true, shape [1,N] becomes [N]. Defaults to false.
health-check-enabled
 false Boolean Whether to enable periodic health checks for the Triton server. When enabled, the health checker will periodically call /v2/health/live endpoint to verify server availability. Defaults to false.
health-check-interval
 30 s Duration Interval between health check requests. Shorter intervals provide faster failure detection but increase server load. Defaults to 30 seconds. Only effective when health-check-enabled is true.
model-name
 (none) String Name of the model to invoke on Triton server.
model-version
 "latest" String Version of the model to use. Defaults to 'latest'.
priority
 (none) Integer Request priority level (0-255). Higher values indicate higher priority.
sequence-end
 false Boolean Whether this request marks the end of a sequence for stateful models. When true, Triton will release the model's state after processing this request. See Triton Stateful Models for more details.
sequence-id
 (none) String Sequence ID for stateful models. A sequence represents a series of inference requests that must be routed to the same model instance to maintain state across requests (e.g., for RNN/LSTM models). See Triton Stateful Models for more details.
sequence-start
 false Boolean Whether this request marks the start of a new sequence for stateful models. When true, Triton will initialize the model's state before processing this request. See Triton Stateful Models for more details.
timeout
 30 s Duration HTTP request timeout (connect + read + write). This applies per individual request and is separate from Flink's async timeout. Defaults to 30 seconds.
