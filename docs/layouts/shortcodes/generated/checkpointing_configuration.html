Key Default Type Description
state.backend.incremental
 false Boolean 选择是否启用 State Backend 创建增量 checkpoint（如果可能）。对于增量 checkpoint，只会存储与上一个 checkpoint 的差异，而不是完整的 Checkpoint State。一旦启用，Web UI 中显示的 State 大小或从 REST API 获取的 State 大小仅表示增量 checkpoint 大小，而不是完整的 checkpoint 大小。某些 State Backend 可能不支持增量 checkpoint 并忽略此选项。
state.backend.local-recovery
 false Boolean 此选项配置了此 State Backend 的本地恢复。默认情况下，本地恢复处于停用状态。目前，本地恢复仅适用于keyed state backend（包括 EmbeddedRocksDBStateBackend 和 HashMapStateBackend）。
state.checkpoint-storage
 (none) String 用于 Checkpoint State 的 Checkpoint storage 实现。
可以通过其简称或 CheckpointStorageFactory 的类名来指定实现。如果指定了工厂，将通过其无参构造函数实例化，并调用其 CheckpointStorageFactory#createFromConfig(ReadableConfig, ClassLoader)方法。
已识别的简称包括 'jobmanager' 和 'filesystem'。
state.checkpoints.dir
 (none) String 在 Flink 支持的文件系统中，用于存储 Checkpoint 的数据文件和元数据的默认目录。存储路径必须对所有参与的进程/节点（即所有 TaskManagers 和 JobManagers）可访问。
state.checkpoints.num-retained
 1 Integer 要保留的完成的 Checkpoint 的最大数量。
state.savepoints.dir
 (none) String Savepoint 的默认目录。用于将 Savepoint 写入文件系统的 State Backend（HashMapStateBackend、EmbeddedRocksDBStateBackend）。
state.storage.fs.memory-threshold
 20 kb MemorySize State data files的最小值。所有小于此大小的 state chunks 都会直接存储在根 checkpoint 元数据文件中。此配置的最大内存阈值为1MB。
state.storage.fs.write-buffer-size
 4096 Integer 用于写入文件系统的 checkpoint streams 的写缓冲区的默认大小。实际的写缓冲区大小确定为此选项值和选项 'state.storage.fs.memory-threshold' 值的较大者。
taskmanager.state.local.root-dirs
 (none) String 用于本地恢复存储文件 state 的根目录的配置参数。当前本地恢复仅适用于 keyed state backend。如果未配置，它将默认为<WORKING_DIR>/localState。可以通过process.taskmanager.working-dir 进行配置。
