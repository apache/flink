Key Default Type Description
cluster.intercept-user-system-exit
 DISABLED

Enum

Possible values: [DISABLED, LOG, THROW]		 Flag to check user code exiting system by terminating JVM (e.g., System.exit())
  • DISABLED - No check is enabled, that is allowing exit without any action
  • LOG - Log exit attempt with stack trace but still allowing exit to be performed
  • THROW - Throw exception when exit is attempted disallowing JVM termination
cluster.processes.halt-on-system-exit
 false Boolean Whether processes should halt on system exit instead of performing a graceful shutdown. In some environments (e.g. Java 8 with the G1 garbage collector), a regular graceful shutdown can lead to a JVM deadlock. See FLINK-16510 for details.
